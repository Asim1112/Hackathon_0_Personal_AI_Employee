# SKILL_Weekly_CEO_Briefing — Agent Skill

## Purpose
Generate a comprehensive weekly CEO briefing every Sunday at 23:00.
Synthesise data from: Odoo accounting, completed tasks, social analytics, pipeline,
and outstanding items into a single `Briefings/CEO_BRIEFING_<YYYY-MM-DD>.md` report.

This is the Gold tier's flagship output — the document that makes the AI Employee
feel like a real business intelligence layer, not just a task processor.

---

## When to Invoke

- Every Sunday at 23:00 (triggered by `orchestrator.py`)
- User requests "generate the weekly briefing" or "what's the week's summary"
- Before an important business meeting

---

## Pre-Conditions: Ensure Fresh Accounting Data

Before generating the briefing, accounting data must be current. Check
`Accounting/Current_Month.md` frontmatter field `updated`.

If it is **older than 24 hours**, first:
1. Invoke `SKILL_Odoo_Accounting` to create a `sync_accounting` action in `Pending_Approval/`
2. Write a note to the briefing: *"Accounting data is being refreshed — sync pending approval."*
3. Continue with last known data

If data is fresh (< 24h old), read `Accounting/Current_Month.md` directly.

---

## Data Sources to Read

Read ALL of these before writing the briefing:

1. `Business_Goals.md` — monthly revenue target, KPIs, active projects, rate card
2. `Accounting/Current_Month.md` — Odoo revenue summary, invoices, overdue
3. `Social_Analytics/Twitter_Summary.md` — follower growth, post performance
4. `Social_Analytics/Facebook_Instagram_Summary.md` — reach, engagement
5. `Dashboard.md` — overall vault state, pending approvals, queue status
6. `Done/` folder — scan for this week's completed tasks (files modified in last 7 days)
7. `Pending_Approval/` — any items awaiting human decision
8. `Plans/` and `In_Progress/` — active multi-step projects

## After Writing the Briefing

1. Save to `Briefings/CEO_BRIEFING_<YYYY-MM-DD>.md`
2. Update `Dashboard.md` → **Recent CEO Briefings** table with new entry
3. Log to `Logs/YYYY-MM-DD.json`:
   ```json
   {
     "action_type": "ceo_briefing_generated",
     "target": "CEO_BRIEFING_<YYYY-MM-DD>.md",
     "skill_used": "SKILL_Weekly_CEO_Briefing",
     "approval_status": "not_required",
     "result": "success",
     "notes": "Weekly briefing for week ending <date>"
   }
   ```
4. Output `TASK_COMPLETE`

---

## Quality Guidelines

- Be concrete with numbers — never write "revenue was good" when you have the figure
- Flag risks clearly with 🔴 — the CEO should never be surprised on Monday morning
- Keep Executive Summary to 5 bullets max — ruthlessly prioritise
- If data is missing (e.g., Odoo sync hasn't run), say so explicitly with the timestamp of last sync
- Do not hallucinate metrics — if you don't have a number, write "data pending sync" not an estimate
