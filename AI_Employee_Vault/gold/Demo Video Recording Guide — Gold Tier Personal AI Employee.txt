  ✅ Twitter/X — 5 API Keys Required

  Twitter uses OAuth 1.0a — you need keys for both your App and your specific user account. All 5 are required; the MCP server exits immediately if any are missing.

  ┌─────────────────────────────┬────────────────────────────────────────────┬────────────────────────────────────────────────────────────────────────────┐
  │         .env Field          │                 What It Is                 │                              Where to Find It                              │
  ├─────────────────────────────┼────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────┤
  │ TWITTER_API_KEY             │ Your App's Consumer Key                    │ Twitter Developer Portal → Project → App → Keys and Tokens → Consumer Keys │
  ├─────────────────────────────┼────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────┤
  │ TWITTER_API_SECRET          │ Your App's Consumer Secret                 │ Same page as above                                                         │
  ├─────────────────────────────┼────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────┤
  │ TWITTER_ACCESS_TOKEN        │ Your account's Access Token                │ Same page → Authentication Tokens → Access Token and Secret                │
  ├─────────────────────────────┼────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────┤
  │ TWITTER_ACCESS_TOKEN_SECRET │ Your account's Access Token Secret         │ Same page as above                                                         │
  ├─────────────────────────────┼────────────────────────────────────────────┼────────────────────────────────────────────────────────────────────────────┤
  │ TWITTER_BEARER_TOKEN        │ App-level read token (used by the watcher) │ Same page → Bearer Token                                                   │
  └─────────────────────────────┴────────────────────────────────────────────┴────────────────────────────────────────────────────────────────────────────┘

  How to get them:
  1. Go to developer.twitter.com and sign in with the account that will be posting
  2. Open your Project → App → Keys and Tokens tab
  3. If you haven't generated Access Token/Secret yet, click Generate under Authentication Tokens
  4. Copy all 5 values into .env

  ⚠️ Access Token type matters. Must be generated with Read and Write permission. To change: Developer Portal → App → Settings → User authentication settings → App
  permissions → set to "Read and Write".

  ---
  ✅ Facebook — 2 Credentials Required

  Facebook posting uses the Meta Graph API v19.0. You post to a Facebook Page (not a personal profile).

  ┌───────────────────────┬────────────────────────────────────────────────────────┬─────────────────────────────────┐
  │      .env Field       │                       What It Is                       │        Where to Find It         │
  ├───────────────────────┼────────────────────────────────────────────────────────┼─────────────────────────────────┤
  │ FACEBOOK_PAGE_ID      │ The numeric ID of your Facebook Page                   │ Facebook Page → About → Page ID │
  ├───────────────────────┼────────────────────────────────────────────────────────┼─────────────────────────────────┤
  │ FACEBOOK_ACCESS_TOKEN │ A Page Access Token with pages_manage_posts permission │ Graph API Explorer (see below)  │
  └───────────────────────┴────────────────────────────────────────────────────────┴─────────────────────────────────┘

  Getting a Page Access Token:
  1. Go to Meta for Developers → Tools → Graph API Explorer
  2. Select your App in the top-right dropdown
  3. Click Generate Access Token → grant all pages permissions
  4. In the left panel, change token type to Page Access Token and select your Page
  5. For a demo, a short-lived token (1 hour) is fine — generate it 30 minutes before recording
  6. Copy into FACEBOOK_ACCESS_TOKEN in .env

  ---
  ✅ Instagram — 1 Credential Required

  Must be a Business or Creator account linked to your Facebook Page.

  ┌──────────────────────┬───────────────────────────────────────────────────┬────────────────────────────────┐
  │      .env Field      │                    What It Is                     │        Where to Find It        │
  ├──────────────────────┼───────────────────────────────────────────────────┼────────────────────────────────┤
  │ INSTAGRAM_ACCOUNT_ID │ The numeric ID of your Instagram Business account │ Graph API Explorer (see below) │
  └──────────────────────┴───────────────────────────────────────────────────┴────────────────────────────────┘

  Finding your Instagram Account ID:
  In Graph API Explorer with your Page Access Token active, call:
  GET /{your-page-id}?fields=instagram_business_account
  The response contains "instagram_business_account": {"id": "17841..."} — that number is your ID.

  ⚠️ Instagram posts require an image_url. For the demo, the Odoo and Twitter items are stronger HITL demonstrations.

  A4 — Vault State Verification

  Step 1 — Run the Reset

  cd "F:\Hackathon 0 Personal AI Employee\AI_Employee_Vault\gold"
  uv run python reset_demo.py

  Clears Needs_Action/, Inbox/, Pending_Approval/, Approved/ → archives stale files → writes 5 fresh demo files → resets Dashboard.md. Does NOT touch Done/, Logs/, or
  Briefings/.

  Step 2 — Generate the CEO Briefing (first time only)

  Takes 2–3 minutes. Survives all subsequent resets — run it once.

  Step 3 — State Check

  uv run python -c "
  from pathlib import Path
  v = Path('.')
  checks = [
    ('Needs_Action has 5 files',    lambda: len(list((v/'Needs_Action').glob('*.md'))) == 5),
    ('Inbox is empty',              lambda: len(list((v/'Inbox').glob('*.md'))) == 0),
    ('Pending_Approval is empty',   lambda: len(list((v/'Pending_Approval').glob('*.md'))) == 0),
    ('Approved is empty',           lambda: len(list((v/'Approved').glob('*.md'))) == 0),
    ('Done has history',            lambda: len(list((v/'Done').glob('*.md'))) > 0),
    ('Logs has audit files',        lambda: len(list((v/'Logs').glob('*.json'))) > 0),
    ('Briefings has CEO briefing',  lambda: len(list((v/'Briefings').glob('*.md'))) > 0),
    ('Dashboard.md exists',         lambda: (v/'Dashboard.md').exists()),
    ('CLAUDE.md exists',            lambda: (v/'CLAUDE.md').exists()),

  all_ok = True
  for name, fn in checks:
      r = fn()
      print('[OK]' if r else '[!!]', name)
      if not r: all_ok = False
  print()
  print('READY TO RECORD' if all_ok else 'NOT READY — fix issues above')
  "

  Orchestrator Stops Mid-Run

  1. Check terminal output — it will show which skill was running
  2. Check Logs/<today>.json — find the last logged entry
  3. Check Needs_Action/ — unprocessed files are still there
  4. Fix: uv run python reset_demo.py then uv run python orchestrator.py --cron

  Files Not Moving After Approval

  1. MCP not running — check terminal tab, restart if crashed
  2. Wrong folder — files must be in Approved/ root, not a subfolder
  3. Wrong frontmatter — type: and send_via_mcp: must match exactly (see CLAUDE.md cheat sheet)
  4. Windows BOM encoding — PowerShell utf8 adds BOM, breaks YAML parser (type becomes type=?). Use Python or VS Code (LF, no BOM) to create/edit files
  5. chokidar delay — wait 2–5 seconds after moving file before assuming it failed
